IB TABLES (supabase/migrations/117_ib_management.sql)
====================================================

1. ib_partners - The IB/affiliate
   - id, organization_id, name, referral_code (unique per org)
   - agreement_type: salary, cpa, lot_rebate, revenue_share, hybrid
   - agreement_details (JSONB), status, contact_email, contact_phone
   
   Extended fields (migration 119):
   - company_name, website, telegram, whatsapp, instagram, twitter, linkedin
   - preferred_payment_method, iban, crypto_wallet_address, crypto_network
   - contract_start_date, contract_end_date, logo_url

2. ib_referrals - Clients referred by IB
   - id, organization_id, ib_partner_id, client_name
   - ftd_date, ftd_amount, is_ftd, lots_traded, status, notes

3. ib_commissions - Commission record
   - id, organization_id, ib_partner_id, period_start, period_end
   - calculated_amount, override_amount, final_amount (generated)
   - status: draft, confirmed, paid
   - breakdown (JSONB)
   - RPC: calculate_ib_commission(partner_id, start, end)

4. ib_payments - Payment to IB
   - id, organization_id, ib_partner_id, ib_commission_id (nullable)
   - amount, currency, register (USDT/NAKIT_TL/NAKIT_USD/TRX)
   - payment_date, payment_method, reference
   
   *** AUTO-TRIGGER: INSERT -> creates accounting_entries row ***

AGREEMENT TYPES & CALCULATION
=============================
1. Salary: Fixed amount per period
2. CPA: FTD count × cpa_amount
3. Lot Rebate: total_lots × rebate_per_lot
4. Revenue Share: total_revenue × (revshare_pct / 100)
5. Hybrid: Sum of multiple components

CONNECTION POINT: accounting_entries
====================================
Both systems feed into accounting_entries:

FROM IB PAYMENTS:
- ib_payments INSERT -> auto-creates accounting_entries
- entry_type='ODEME', direction='out'
- Linked via ib_payment_id FK

FROM TRANSFERS:
- Would need transfer_id field in accounting_entries
- (current schema may have this - needs verification)

accounting_entries fields:
- id, organization_id, entry_type, direction, amount, currency
- register, description, entry_date, ib_payment_id
- Also: hr_bulk_payment_id, hr_employee_id, hr_payment_id, etc

NO EXISTING IB-TRANSFER INTEGRATION
===================================

✗ Transfers don't reference IB partners
✗ Transfer form has no IB field
✗ Creating an ib_referral doesn't create a transfer
✗ Creating a transfer doesn't create an ib_referral
✗ Revenue share commission can't auto-calc from transfers
✗ No way to know if a transfer is from an IB referral


POTENTIAL INTEGRATION OPTIONS
=============================

A) Add ib_partner_id to transfers
   - Direct attribution
   - Schema migration required
   
B) Use referral code in transfers
   - No schema change (store in notes/meta)
   - Looser coupling
   
C) Auto-link via FTD detection
   - If transfer matches ib_referral's ftd_date/amount
   - Complex matching logic
