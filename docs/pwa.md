# PWA & Mobile

**Status:** Living spec · reflects codebase as of `main` on 2026-04-24
**Scope:** Installable web app, service worker, offline, mobile layout conventions
**Related:** [design-system/accessibility.md §4.4](./design-system/accessibility.md#44-pwa-standalone), [design-system/patterns.md §1](./design-system/patterns.md#1-page-scaffold)

## 1. Stack

- **Build plugin:** `vite-plugin-pwa` ([vite.config.ts](../vite.config.ts))
- **Service worker strategy:** Workbox (default `generateSW` preset + our runtime rules)
- **Register type:** `'prompt'` — SW updates surface a toast, user clicks to apply
- **Manifest:** generated by the plugin from config

## 2. Manifest

Declared in `vite.config.ts` under the `VitePWA` plugin options:

| Field | Value |
|---|---|
| `name` | `'PipLinePro'` |

**Icons:** PNG only. No SVG manifest icons. Maskable icon variants required for Android A2HS chrome.

## 3. Service worker

### 3.1 Registration

In [src/main.tsx](../src/main.tsx):

```ts
import { registerSW } from 'virtual:pwa-register'
```

Dynamic import (code-split). Subscribes to update / need-refresh events and pipes them into [`src/lib/pwaUpdateController.ts`](../src/lib/pwaUpdateController.ts).

### 3.2 Update prompt

[`src/components/PwaUpdatePrompt.tsx`](../src/components/PwaUpdatePrompt.tsx) — small toast UI that appears when a new SW is waiting. Click → reload to activate. Dismiss → postponed until next load.

### 3.3 Runtime caching

Default Workbox preset caches:
- JS / CSS / HTML assets (cache-first with revalidate).
- Images from same-origin (cache-first).

**No** custom runtime rules for Supabase API calls — those go network-first via fetch. Offline-capable reads would require IndexedDB caching of query results; not implemented.

### 3.4 Offline fallback

[`public/offline.html`](../public/offline.html) — branded "you're offline" page. Served by the SW when a network navigation fails and the target isn't in cache.

## 4. Mobile layout conventions

### 4.1 Bottom navigation

[`src/components/BottomNav.tsx`](../src/components/BottomNav.tsx). Visible only on `<md` (768px). Five items:
- Dashboard
- Transfers
- HR / Members (role-based — HR for god, Members for others)
- AI (`/future`)
- More (opens sidebar drawer via `useSidebar().toggleSidebar()`)

Styling:
- Fixed bottom, `h-16`, `md:hidden`.
- Glass-morphism background (`bg-bg1/80 backdrop-blur`).
- Safe-area padding: `pb-[env(safe-area-inset-bottom)]`.

**Consequence:** every page adds `pb-20 md:pb-6` to its outer wrapper so content isn't hidden behind the nav — see [design-system/patterns.md §1](./design-system/patterns.md#1-page-scaffold).

### 4.2 Header

Compact on mobile: `h-12 md:h-14`. Hides search, online-count, and separators. Adds safe-area top padding in standalone mode:

```css
@media all and (display-mode: standalone) {
  header { padding-top: env(safe-area-inset-top, 0px); }
}
```

See [src/styles/index.css:539–543](../src/styles/index.css#L539-L543).

### 4.3 Safe areas

CSS vars exposed in [src/styles/index.css:505–509](../src/styles/index.css#L505-L509):

```css
--safe-top:    env(safe-area-inset-top, 0px)
--safe-bottom: env(safe-area-inset-bottom, 0px)
```

Use in full-bleed fixed UI (bottom nav, top banners).

### 4.4 Touch targets

Enforced globally for coarse pointers ([index.css:513–531](../src/styles/index.css#L513-L531)):

```css
@media (pointer: coarse) {
  button, a, [role='button'], [role='tab'], [role='menuitem'],
  input[type='checkbox'], input[type='radio'] {
    min-height: 44px;
    min-width: 44px;
  }
}
```

Opt-out: add `.inline-link` or `[data-compact-touch]` to the element.

### 4.5 iOS input zoom prevention

[index.css:550–556](../src/styles/index.css#L550-L556): inputs get `font-size: max(16px, 1em)` on iOS to prevent the zoom-on-focus behavior.

### 4.6 Momentum scroll

`.scroll-touch` utility ([index.css:558–562](../src/styles/index.css#L558-L562)) — adds `-webkit-overflow-scrolling: touch` + `overscroll-behavior-y: contain`.

### 4.7 Press feedback

`.press-scale` utility — 3% scale-down on active for touch pointers. Buttons already have `active:scale-[0.97]` inline.

## 7. Install flow

### 7.1 Chrome/Edge (desktop + Android)

Standard Add to Home Screen prompt. Triggered by browser when manifest + SW criteria are met.

### 7.2 iOS Safari

Manual: Share → Add to Home Screen. No programmatic prompt. The PWA runs in `standalone` display mode once installed.

### 7.3 What changes in standalone mode

- Browser chrome gone.
- `display-mode: standalone` CSS media query matches.
- Safe-area insets become non-zero on notched devices.
- Hash-based links behave like full-page navigations.

## 8. Known gaps

- **No push notifications.** No VAPID key, no push subscription flow. Add if notification use case emerges.
- **No background sync.** Writes that happen offline are lost (forms don't buffer). For low-latency forms (QR check-in), consider Workbox background-sync queue.
- **No offline data reads.** React Query cache is in-memory only. A service-worker-backed IndexedDB cache for recent `/transfers` / `/accounting` queries would make the app useful offline.
- **No installability analytics.** We don't track how many users install vs access via browser tab. Consider an event on `beforeinstallprompt`.
- **Update prompt is eager.** Any SW update surfaces the toast immediately, even mid-task. Consider debouncing: show the toast only when idle.
- **No versioning of the SW cache.** Cached assets get revalidated via hash-in-filename (Vite default), but the cache name itself isn't versioned. If the SW strategy changes, users may see stale caches until manual clear.
- **Icons don't cover every size.** We ship 192 and 512. Some Android launchers prefer 144, 72, 48. Generate via a manifest-icon tool if adoption matters.
- **No iOS splash screens.** iOS uses a static first-paint image when the PWA is launched from Home Screen. We don't ship per-device splash PNGs. Results in a brief flash.
- **No `display_override`.** The manifest uses `display: 'standalone'` without `display_override`. Newer Android browsers may prefer `'window-controls-overlay'` for richer desktop PWA chrome.
